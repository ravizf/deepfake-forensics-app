API Docs

Developer-facing endpoints for the SnapTrace prototype.

Use the public prediction API for demo integrations, then move to the authenticated endpoints for analyst workflows and case retrieval.

Public API{% if public_api_enabled %}Enabled{% else %}Disabled{% endif %}
Main Predict RoutePOST /predict
Upload Fieldmultipart: file

POST /predict

Public prototype endpoint for quick demo integrations.

curl -X POST "{{ request.url_root.rstrip('/') }}/predict" \
  -F "file=@sample.jpg"

Response fields

  • prediction
  • confidence
  • risk_score
  • risk_level
  • analysis_engine
  • detection_mode
  • evaluation_status
  • face_detected
  • metadata_found
  • metadata_check
  • analysis_reasons
  • heatmap_url

Additional Endpoints

  • POST /api/public-analyze returns the full public analysis payload.
  • GET /health returns a simple service heartbeat.
  • GET /api/health returns detector status and public API state.
  • POST /api/analyze is the authenticated upload API for signed-in analysts.
  • GET /api/model-status exposes the active detector descriptor for logged-in users.
